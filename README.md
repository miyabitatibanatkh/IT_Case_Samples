# IT Case Samples

このリポジトリは、実際の企業システムでよく使われる業務機能を練習するためのものです。

単なる Demo を作ることが目的ではなく、実務に近い開発手順で、1つのシステムにつき1つのブランチを作成し、順番に完成させていきます。

- フロントエンド：Vue.js
- バックエンド：.NET Web API
- データベース：PostgreSQL Server
- バージョン管理：Git + GitHub

各練習プロジェクトは、日本企業でよく見られる業務システムを想定しています。完成後は、転職活動用のポートフォリオや、面接で実務力を説明するための材料として活用できます。

## 練習方法

メインブランチには、説明資料や共通情報だけを置きます。

各システムは、それぞれ専用の Git ブランチを作成して練習します。

```text
main
feature/01-user-management
feature/02-leave-request
feature/03-expense-management
feature/04-time-tracking
feature/05-inventory-management
feature/06-order-management
feature/07-meeting-room-booking
feature/08-crm
feature/09-email-notification
feature/10-asset-management
```

1つのブランチでは、対応する1つのシステムだけを作成します。これにより、各プロジェクトの開発過程が分かりやすくなり、あとから個別に見直し、修正、説明しやすくなります。

## 各プロジェクトの基本手順

各システムは、基本的に以下の流れで進めます。

1. 業務要件を整理する
2. データベース設計を行う
3. バックエンド API を作成する
4. フロントエンド画面を作成する
5. フロントエンドとバックエンドを接続する
6. バリデーションとエラーハンドリングを追加する
7. 主要機能をテストする
8. README とコミット履歴を整理する

最初から完璧なシステムを作る必要はありません。まずは最小限動くものを作り、その後で権限管理、ページング、検索、ファイルアップロード、承認フロー、メール通知、集計機能などを少しずつ追加していきます。

- 管理者はすべての予約を取り消せる

練習ポイント：

- 時間判定

- 重複チェック

## 推奨学習順序

### 第一段階：基本的な業務システム

1. ユーザー管理システム
2. 工数管理システム
3. 会議室予約システム

ここまで完成すると、初級エンジニアに必要な CRUD、ログイン認証、検索、日付処理、簡単な権限管理を一通り理解できます。

### 第二段階：承認と通知

1. 休暇申請システム
2. 経費精算システム
3. メール通知システム

ここまで完成すると、企業システムで重要な承認フロー、ステータス遷移、ファイルアップロード、通知機能を理解できます。

### 第三段階：複雑な業務関係

1. 注文管理システム
2. 在庫管理システム
3. 顧客管理システム
4. 資産管理システム

ここまで完成すると、製造業、EC、派遣、金融系バックオフィス、社内システムでよく出てくる業務ロジックを幅広く練習できます。

## Codex との進め方

新しいプロジェクトを始めるときは、次のように依頼します。

```text
ユーザー管理システムを始めます。ブランチ作成から細かく一緒に進めてください。
```

Codex は以下の流れで一緒に進めます。

1. 現在の Git 状態を確認する
2. 対応する機能ブランチを作成する
3. データベースを設計する
4. バックエンドプロジェクトを作成する
5. API を実装する
6. Swagger または API ツールで動作確認する
7. Vue フロントエンドを作成する
8. API と画面を接続する
9. 全体の動作を確認する
10. コミット履歴と README を整理する

各ステップでは、なぜその作業を行うのか、どのコマンドを実行するのか、ファイルをどこに置くのか、コードが何をしているのか、エラーが出た場合にどう確認するのかを説明しながら進めます。
